MySql

1.初期开源阶段

2.Sun MySQL阶段

3.Oracle MySQL阶段

MySQL是一个 关系型数据库管理系统，在1996年发布了1.0版本。到了同年10月发布了MySQL 3.11.1版本（没有2.x版本）。1999年，瑞典MySQL AB公司成立，Monty牵头开发BDB引擎，因BDB支持事务。所以MySQL也开始支持事务处理了，并且提供众多编程语言接口。

版本迭代

##Version 3.23（2001）

版本3.23被认为是MySQL真正诞生的时刻，其开始获得广泛使用。主要改进是引入MYISAM替代了ISAM引擎，引入了全文索引和复制。同年，INnodb创始人向MySQL提议希望支持他的innodb存储引擎，这个存储引擎支持行级锁，支持事务，该引擎之后被证明是最为成功的MySQL存储引擎。

##Version 4.0（2003）

支持最新语法，比如UNION和多表DELETE语法。重写复制，在备库使用了两个线程来实现复制。Innodb成为标准配备，引入查询缓存技术。

Version 4.1（2005）

引入了更多新的语法，并且开始支持UTF-8字符集。

Version 5.0（2006）

引入企业级特性，视图、触发器、存储过程和存储函数。引入新的federated等引擎。

Version 5.1（2008）

08年，SUN以10亿美元收购MySQL AB公司。在这期间MySQL社区都担心Oracle公司会不允许MySQL使用Innodb引擎了，因为在05年，Oracle受够了Innodb存储引擎。

Sun公司在之后对其进行了大量的推广，优化，BUG修复。同年并发布了MySQL 5.1版本，引入了分区、基于行的复制，移除BDB引擎。

Version 5.5（2010）

09年，Oracle公司以73亿美元收购Sun公司。自此MySQL进入Oracle时代。在Oracle收购了MySQL后，开源社区皆十分担心MySQL是否会继续维持GPL协议。

而Oracle在发布5.5版本的时候也承诺5年内Oracle公司会对MySQL继续大力发展。

而收购后主要进行了如下改善：

1.对Window更友好

2.MySQL5.5.8 默认DBMS是InnoDB plugin，用Metadata LOcking Framework替换原来的LOCK_open mutex使得MySQL5.5在四核以上的服务器中性能得到提高。

3.MySQL5.5将支持半同步（seme-sync replication MySQL半同步），在MySQL高可用方案上有个更实际可行的办法。

4.增加了performance_schema库，包含了一些可测量的性能指标的增强，增加了复制、认证和审计API。InnoDB Plugin支持数据压缩存储，节约存储，提高内存命中率。

5.InnoDB Plugin支持adaptive flush checkpoint, 可以在某些场合避免数据库出现突发性瓶颈，Support for Native AIO on Linux。

6.Multi Rollback Segments原来InnoDB只有一个Segment，同时只支持1023的并发。先已扩充到128个Segments，从而解决了高并发的限制。

Version 5.6（2012）

12年，5.6公布了第一个RC版本。13年，5.6第一个GA版本。5.6公布了一系列新特性，主要对安全、数据类型、InnoDB增强，分区的改进，复制的改进，增强了performance schema数据库，优化器的改进等等。
